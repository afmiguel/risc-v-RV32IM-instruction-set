SRAI - Deslocamento Aritmético à Direita com Imediato

Sintaxe

srai rd, rs1, shamt

Descrição

Desloca aritmeticamente o valor no registrador rs1 para a direita pelo número de bits especificado no campo imediato shamt (shift amount, 5 bits), preservando o bit de sinal (MSB) nas posições vazias à esquerda, e armazena o resultado em rd.

Formato e Codificação

Formato: I-type
Opcode: 0010011
funct3: 0x5
shamt: 5 bits (0-31)

Operação

rd = (signed)rs1 >> shamt

Exemplos de Uso

Exemplo 1: Dividir número com sinal por potência de 2

# Dividir -32 por 8 (2^3) li x5, -32 # x5 = -32 srai x6, x5, 3 # x6 = -4 (-32 / 8)
SRAI preserva o sinal durante o deslocamento, tornando-o ideal para divisão rápida de números com sinal por potências de 2.

Exemplo 2: Comparar SRAI e SRLI com número negativo

# Demonstrar diferença entre SRAI e SRLI li x10, -64 # x10 = 0xFFFFFFC0 = -64 srai x11, x10, 2 # x11 = -16 = 0xFFFFFFF0 (sinal preservado) srli x12, x10, 2 # x12 = 1073741808 = 0x3FFFFFF0 (preenchido com 0)
Este exemplo ilustra como SRAI mantém o valor negativo preservando o bit de sinal, enquanto SRLI trata o valor como sem sinal.

Exemplo 3: Implementar divisão por 2 com arredondamento correto

# Dividir número com sinal por 2 li x20, -7 # x20 = -7 srai x21, x20, 1 # x21 = -4 (-7 / 2 arredondado para -infinito)
SRAI implementa divisão inteira que arredonda em direção ao infinito negativo. Para -7 / 2, o resultado é -4 (não -3).

Observações

  • O campo shamt é de 5 bits, permitindo valores de 0 a 31
  • Mais eficiente que SRA quando o valor de deslocamento é constante
  • O bit de sinal (MSB) é replicado para as posições vazias à esquerda
  • Preserva o sinal do número original
  • Diferente de SRLI que sempre preenche com zeros
  • Arredonda em direção ao infinito negativo (floor division)
