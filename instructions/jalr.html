JALR - Salto e Liga via Registrador (Jump and Link Register)

Sintaxe

jalr rd, rs1, imm

Descrição

Armazena o endereço de retorno (PC+4) no registrador rd, então salta para o endereço calculado adicionando o valor imediato imm ao valor no registrador rs1. O bit menos significativo do endereço calculado é zerado (alinhamento de 2 bytes).

Formato e Codificação

Formato: I-type
Opcode: 1100111
funct3: 0x0
Imediato: 12 bits com sinal

Operação

temp = PC + 4 PC = (rs1 + imm) & ~1 rd = temp

Exemplos de Uso

Exemplo 1: Retorno de função (ret)

funcao: addi x10, x10, 5 jalr x0, ra, 0 # retorna (equivalente a "ret") # PC = ra + 0, não salva endereço
Uso mais comum: retornar de função. A pseudo-instrução "ret" é traduzida para "jalr x0, ra, 0".

Exemplo 2: Chamada indireta via ponteiro de função

la x5, funcao_ptr # carrega endereço do ponteiro lw x6, 0(x5) # lê endereço da função li x10, 100 # argumento jalr ra, x6, 0 # chama função via ponteiro # salva retorno em ra
Permite chamadas indiretas através de ponteiros de função, essencial para callbacks e tabelas de funções.

Exemplo 3: Salto para endereço calculado com offset

la x5, tabela_saltos slli x6, x10, 2 # multiplica índice por 4 add x5, x5, x6 # calcula endereço na tabela lw x7, 0(x5) # lê endereço destino jalr x0, x7, 0 # salta sem salvar retorno
Implementação de switch/case ou tabela de despacho (jump table).

Observações

  • Esta é a única instrução que permite salto para endereço calculado dinamicamente
  • O endereço alvo é (rs1 + imm) com o bit 0 zerado (alinhamento)
  • Essencial para implementar retornos de função, ponteiros de função e bibliotecas dinâmicas
  • Se rd = x0, não salva endereço de retorno (usado em "ret" e "jr")
  • Se rd = ra e rs1 = ra, permite ajuste fino do endereço de retorno
  • Alcance ilimitado (qualquer endereço de 32 bits via rs1)
  • O imediato permite pequenos ajustes (±2048 bytes) no endereço base
  • Fundamental para Position-Independent Code (PIC)
