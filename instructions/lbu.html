LBU - Carregar Byte sem Sinal (Load Byte Unsigned)

Sintaxe

lbu rd, imm(rs1)

Descrição

Carrega um byte (8 bits) da memória para o registrador de destino rd. O endereço de memória é calculado somando o deslocamento imediato imm ao valor no registrador base rs1. O byte carregado é então estendido com zeros para 32 bits antes de ser armazenado em rd.

Formato e Codificação

Formato: I-type
Opcode: 0000011
funct3: 0x4

Operação

rd = zero_extend(mem[rs1 + imm])

Exemplos de Uso

Exemplo 1: Ler dados de um buffer

.data buffer: .byte 0x1A, 0x80, 0xFF .text la t0, buffer lbu t1, 1(t0) # Carrega o segundo byte (0x80) # t1 conterá 0x00000080 (128 decimal)
`lbu` é usado quando o byte representa um valor que deve ser tratado como positivo (0-255), como um pixel de uma imagem em escala de cinza ou um dado de um sensor.

Exemplo 2: Diferença para LB

# Suponha que o endereço 0x1000 contém o byte 0x80 (-128) li t0, 0x1000 lbu t1, 0(t0) # t1 = 0x00000080 (128) - Extensão com zero lb t2, 0(t0) # Para comparação, lb resultaria em 0xFFFFFF80 (-128)
Este exemplo mostra a diferença crucial: `lbu` preenche os 24 bits superiores com 0 (extensão de zero), tratando o byte como um número sem sinal. `lb` preencheria com 1, pois o bit 7 do byte é 1.

Observações

  • O deslocamento imm é um valor com sinal de 12 bits (faixa de -2048 a +2047).
  • A extensão com zero torna `lbu` ideal para carregar bytes que representam valores sem sinal (0 a 255).
  • Para carregar bytes com sinal (tratando-os como valores de -128 a 127), use a instrução lb.
  • O endereço de memória (rs1 + imm) não precisa estar alinhado.
