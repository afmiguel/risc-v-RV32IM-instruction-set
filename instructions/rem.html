REM/REMU - Resto da Divisão

Sintaxe

rem rd, rs1, rs2 # Resto com sinal remu rd, rs1, rs2 # Resto sem sinal

Descrição

REM calcula o resto da divisão com sinal de rs1 por rs2, armazenando o resultado em rd. REMU realiza a mesma operação tratando ambos os operandos como números sem sinal. Faz parte da extensão M.

Formato e Codificação

Formato: R-type
Opcode: 0110011
funct3 (REM): 0x6
funct3 (REMU): 0x7
funct7: 0x01

Operação

REM: rd = (signed)rs1 % (signed)rs2 REMU: rd = (unsigned)rs1 % (unsigned)rs2

Exemplos de Uso

Exemplo 1: Operação módulo com sinal

# Calcular resto da divisão li x5, 17 # x5 = 17 li x6, 5 # x6 = 5 rem x7, x5, x6 # x7 = 2 (17 % 5 = 2)
REM retorna o resto da divisão. O sinal do resultado segue o sinal do dividendo (primeiro operando).

Exemplo 2: Verificar se número é múltiplo

# Verificar se x10 é múltiplo de 4 remu x11, x10, 4 # x11 = x10 % 4 beqz x11, eh_multiplo # se resto = 0, é múltiplo
Uso comum: verificar divisibilidade. Se o resto é zero, o número é divisível pelo divisor.

Exemplo 3: Implementar aritmética circular

# Circular buffer: índice = (índice + 1) % tamanho li x20, 15 # índice atual = 15 addi x20, x20, 1 # incrementar: x20 = 16 li x21, 16 # tamanho do buffer = 16 remu x20, x20, x21 # x20 = 0 (volta ao início)
Operação módulo é essencial para implementar buffers circulares e índices que devem "dar a volta".

Observações

  • Resto por zero NÃO gera exceção - retorna o dividendo (rs1)
  • Para REM, o sinal do resultado é o mesmo do dividendo (rs1)
  • REM: se rs1 = -7 e rs2 = 3, resultado = -1 (não +2)
  • Faz parte da extensão M - requer processador RV32IM ou superior
  • Satisfaz: (rs1/rs2)*rs2 + (rs1%rs2) = rs1
  • Útil para operações módulo, verificação de divisibilidade e índices circulares
