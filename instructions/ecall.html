ECALL - Chamada de Sistema (Environment Call)

Sintaxe

ecall

Descrição

Gera uma exceção de chamada de sistema do ambiente de execução. Transfere controle para o modo de maior privilégio (tipicamente modo M - Machine) para executar serviços do sistema operacional ou runtime. O código da syscall é passado em a7 (x17) e argumentos em a0-a6.

Formato e Codificação

Formato: I-type
Opcode: 1110011
funct3: 0x0
imm: 0x000

Operação

RaiseException(EnvironmentCall) PC = mtvec mepc = PC (da instrução ecall) mcause = 11 (environment call from M-mode)

Exemplos de Uso

Exemplo 1: Imprimir string (RARS/MARS)

la x10, mensagem # a0 = endereço da string li x17, 4 # a7 = 4 (print string) ecall # chama sistema mensagem: .string "Hello, RISC-V!\n"
Em simuladores como RARS, ecall com código 4 imprime string apontada por a0.

Exemplo 2: Ler inteiro do usuário

li x17, 5 # a7 = 5 (read integer) ecall # chama sistema # valor lido retorna em a0 (x10) mv x11, x10 # salva valor em x11
Syscall 5 lê um inteiro da entrada padrão e retorna em a0.

Exemplo 3: Terminar programa

main: # código do programa li x10, 42 # código de saída li x17, 93 # a7 = 93 (exit com código) ecall # termina programa # ou simplesmente li x17, 10 # a7 = 10 (exit - RARS/MARS) ecall # termina programa
Diferentes ambientes usam códigos diferentes para terminar: 93 (Linux), 10 (RARS/MARS).

Observações

  • Mecanismo padrão para interação com sistema operacional ou runtime
  • Código da syscall em a7 (x17), argumentos em a0-a6 (x10-x16)
  • Resultado retorna em a0 (x10) e a1 (x11) se necessário
  • Causa transição de modo de privilégio (User → Machine/Supervisor)
  • O handler de exceção é definido pelo registrador mtvec
  • Códigos de syscall variam entre ambientes (Linux, RARS, bare-metal)
  • Simuladores RARS/MARS têm syscalls simplificadas para I/O
  • Em sistemas bare-metal, pode ser usado para debugger/monitor
  • PC da instrução ecall é salvo em mepc para possível retorno
