SLTIU - Setar se Menor Que, Imediato, sem Sinal (Set if Less Than, Immediate, Unsigned)

Sintaxe

sltiu rd, rs1, imediato

Descrição

Compara o valor no registrador rs1 com o valor imediato, tratando ambos como números sem sinal. Se rs1 for menor que o imediato, armazena 1 no registrador de destino rd. Caso contrário, armazena 0.

Formato e Codificação

Formato: I-type
Opcode: 0010011
funct3: 0x3

Operação

if ((unsigned)rs1 < (unsigned)imediato) then rd = 1 else rd = 0 endif

Exemplos de Uso

Exemplo 1: Verificar um limite superior (sem sinal)

# Suponha que t0 contém um tamanho ou contagem sltiu t1, t0, 2048 # t1 = 1 se t0 < 2048 # O imediato é tratado como um número sem sinal (0 a 4095) # mas a comparação é rs1 < imediato, então o imediato 1 # é um caso especial.
`sltiu` é usado para comparações com constantes quando os valores são inerentemente sem sinal, como tamanhos, contagens ou endereços.

Exemplo 2: Implementando `seqz`

# seqz rd, rs (pseudo-instrução) # é implementado como: sltiu rd, rs, 1 # rd = 1 se rs < 1 (sem sinal), o que só é verdade se rs = 0.
A pseudo-instrução `seqz` (Set if Equal to Zero) é um caso especial de `sltiu`. Comparar se um número sem sinal é menor que 1 é o mesmo que testar se ele é 0.

Observações

  • O valor imediato é um número de 12 bits, que na comparação é estendido com zero para 32 bits.
  • Realiza uma comparação sem sinal. Para comparação com sinal com um imediato, use slti.
  • A instrução `sltiu rd, rs, 0` nunca pode ser verdadeira e sempre resultará em 0, pois nenhum número sem sinal é menor que 0.
  • `sltiu rd, rs, 1` é a implementação canônica da pseudo-instrução `seqz rd, rs`.
