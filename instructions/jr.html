JR - Salto via Registrador (Jump Register)

Sintaxe

jr rs1

Descrição

Salta para o endereço armazenado no registrador rs1. Esta é uma pseudo-instrução que é traduzida para jalr x0, rs1, 0. Não salva endereço de retorno.

Formato e Codificação

Formato: Pseudo-instrução (I-type)
Tradução: jalr x0, rs1, 0
Opcode: 1100111
funct3: 0x0

Operação

PC = rs1 & ~1

Exemplos de Uso

Exemplo 1: Salto indireto simples

la x5, destino # carrega endereço jr x5 # salta para endereço em x5 destino: # código continua aqui
Uso básico para salto indireto quando não precisa salvar endereço de retorno.

Exemplo 2: Implementação de switch/case

# x10 contém o índice do case la x5, tabela_casos slli x6, x10, 2 # multiplica por 4 (tamanho do ponteiro) add x5, x5, x6 # calcula endereço na tabela lw x7, 0(x5) # lê endereço do case jr x7 # salta para case correspondente tabela_casos: .word caso_0 .word caso_1 .word caso_2 caso_0: # código do case 0 j fim_switch caso_1: # código do case 1 j fim_switch caso_2: # código do case 2 j fim_switch fim_switch:
Jump table para implementação eficiente de switch/case com múltiplas opções.

Exemplo 3: Despacho de funções

# Tabela de ponteiros de função la x5, funcoes lw x6, 0(x10) # índice em x10 slli x6, x6, 2 add x5, x5, x6 lw x7, 0(x5) # carrega ponteiro de função jr x7 # chama função sem salvar retorno funcoes: .word func_a .word func_b .word func_c
Útil para tabelas de despacho onde não é necessário retornar ao ponto de chamada.

Observações

  • Esta é uma pseudo-instrução expandida para jalr x0, rs1, 0
  • Não salva endereço de retorno (usa x0 como destino)
  • O endereço em rs1 deve estar alinhado a 2 bytes (bit 0 zerado automaticamente)
  • Diferente de "ret" que assume endereço em ra, "jr" permite qualquer registrador
  • Útil para jump tables e despacho dinâmico
  • Mais flexível que "j" pois o destino é calculado em runtime
  • Não deve ser confundido com "ret" que é especificamente para retorno de função
