FENCE - Barreira de Memória

Sintaxe

fence pred, succ

Descrição

A instrução fence ordena a execução de operações de memória e I/O. Garante que todas as operações de memória especificadas no conjunto `pred` (predecessor), realizadas por esta thread, sejam concluídas antes de qualquer operação de memória no conjunto `succ` (sucessor) da mesma thread.

Formato e Codificação

Formato: I-type
Opcode: 0001111
funct3: 0x0

Operação

# Ordena operações de memória

Exemplos de Uso

Exemplo 1: Sincronização entre threads

# Thread 1: sw a0, 0(t0) # Escreve dados fence w, w # Garante que a escrita seja visível sw a1, 0(t1) # Escreve flag de conclusão # Thread 2: loop: lw t2, 0(t1) # Lê a flag bnez t2, loop # Espera a flag ser setada fence r, r # Garante que leituras subsequentes vejam os dados lw a2, 0(t0) # Lê os dados com segurança
`fence` é crucial em sistemas multicore para garantir que as escritas de uma CPU se tornem visíveis para outras CPUs na ordem correta.

Observações

  • Os operandos `pred` e `succ` são máscaras de 4 bits que especificam os tipos de operação: I (I/O), O (escrita), R (leitura), W (escrita).
  • `fence i, o` garante que leituras de I/O anteriores terminem antes de escritas de memória posteriores.
  • Uma `fence` completa (`fence r,w,o,i, r,w,o,i`) é a barreira mais forte.
  • É uma instrução complexa, geralmente usada em programação de baixo nível, como em drivers ou sistemas operacionais.
