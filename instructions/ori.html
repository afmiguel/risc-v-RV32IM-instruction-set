ORI - Operação OU bit a bit com Imediato

Sintaxe

ori rd, rs1, imm

Descrição

Executa a operação lógica OR bit a bit entre o valor no registrador rs1 e um valor imediato de 12 bits com extensão de sinal, armazenando o resultado no registrador rd.

Formato e Codificação

Formato: I-type
Opcode: 0010011
funct3: 0x6
funct7: N/A

Operação

rd = rs1 | sign_extend(imm[11:0])

Exemplos de Uso

Exemplo 1: Setar bits específicos

# Setar os bits 0, 2 e 4 li x5, 0x100 # x5 = 256 (binário: 100000000) ori x6, x5, 0x15 # x6 = 277 (binário: 100010101)
O ORI é usado para forçar bits específicos para 1. Neste exemplo, os bits nas posições 0, 2 e 4 (máscara 0x15 = 10101) são setados.

Exemplo 2: Criar máscara com bits inferiores

# Garantir que os 4 bits inferiores estejam setados li x10, 0xAB0 # x10 = 2736 ori x11, x10, 0xF # x11 = 0xABF = 2751
Útil para garantir que certos bits estejam sempre em 1, independente do valor original do registrador.

Exemplo 3: Copiar registrador (com OR zero)

# Copiar valor de x5 para x6 ori x6, x5, 0 # x6 = x5 | 0 = x5
Fazer OR com zero não altera o valor, criando uma cópia. Embora menos comum que MV (que é um pseudo-instrução para ADDI), ainda é válido.

Observações

  • O valor imediato é de 12 bits e passa por extensão de sinal para 32 bits
  • Um bit no resultado será 1 se pelo menos um dos operandos tiver 1 naquela posição
  • Comumente usado para setar flags e bits de controle em registradores
  • Útil em programação de dispositivos de I/O e configuração de periféricos
