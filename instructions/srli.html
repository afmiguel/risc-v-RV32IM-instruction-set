SRLI - Deslocamento Lógico à Direita com Imediato

Sintaxe

srli rd, rs1, shamt

Descrição

Desloca logicamente o valor no registrador rs1 para a direita pelo número de bits especificado no campo imediato shamt (shift amount, 5 bits), inserindo zeros nas posições vazias à esquerda, e armazena o resultado em rd.

Formato e Codificação

Formato: I-type
Opcode: 0010011
funct3: 0x5
shamt: 5 bits (0-31)

Operação

rd = (unsigned)rs1 >> shamt

Exemplos de Uso

Exemplo 1: Dividir por potência de 2 (números sem sinal)

# Dividir 100 por 4 (2^2) li x5, 100 # x5 = 100 srli x6, x5, 2 # x6 = 25 (100 / 4)
SRLI é eficiente para divisão por potências de 2 quando o divisor é uma constante conhecida. Deslocar por 2 divide por 4.

Exemplo 2: Extrair byte superior

# Extrair o byte mais significativo li x10, 0xABCD1234 # x10 = 2882343476 srli x11, x10, 24 # x11 = 0xAB = 171
Deslocar 24 bits à direita move o byte mais significativo para os 8 bits menos significativos, isolando-o efetivamente.

Exemplo 3: Processar campos de instrução

# Extrair opcode de uma instrução RISC-V (bits 0-6) # Supondo x20 contém uma instrução codificada srli x21, x20, 0 # não precisa deslocar andi x21, x21, 0x7F # isola os 7 bits do opcode # Ou extrair rd (bits 7-11) srli x22, x20, 7 # move rd para LSB andi x22, x22, 0x1F # isola 5 bits
SRLI é fundamental para decodificar instruções, permitindo extrair campos específicos da codificação de 32 bits.

Observações

  • O campo shamt é de 5 bits, permitindo valores de 0 a 31
  • Mais eficiente que SRL quando o valor de deslocamento é constante
  • Zeros são sempre inseridos à esquerda (deslocamento lógico)
  • Não preserva o sinal - use SRAI para números com sinal
  • Útil para extrair bytes e campos de bits de posições conhecidas
