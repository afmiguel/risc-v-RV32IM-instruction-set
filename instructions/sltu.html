SLTU - Setar se Menor Que, sem Sinal (Set if Less Than, Unsigned)

Sintaxe

sltu rd, rs1, rs2

Descrição

Compara os valores nos registradores rs1 e rs2 como inteiros sem sinal. Se o valor em rs1 for menor que o valor em rs2, armazena 1 no registrador de destino rd. Caso contrário, armazena 0.

Formato e Codificação

Formato: R-type
Opcode: 0110011
funct3: 0x3
funct7: 0x00

Operação

if ((unsigned)rs1 < (unsigned)rs2) then rd = 1 else rd = 0 endif

Exemplos de Uso

Exemplo 1: Comparar ponteiros de memória

la t0, array_inicio la t1, array_fim # Suponha que t2 é o ponteiro atual sltu t3, t2, t1 # t3 = 1 se o ponteiro atual ainda não chegou ao fim
Endereços de memória são sempre positivos, então comparações entre eles devem usar a forma sem sinal (`sltu` e `sltiu`).

Exemplo 2: Diferença para SLT

li t0, -1 # t0 = 0xFFFFFFFF li t1, 1 # t1 = 0x00000001 slt t2, t0, t1 # t2 = 1, porque (signed)-1 < (signed)1 sltu t3, t0, t1 # t3 = 0, porque (unsigned)0xFFFFFFFF > (unsigned)1
Este exemplo clássico demonstra a importância de escolher a instrução correta. Como um número com sinal, -1 é menor que 1. Como um número sem sinal, 0xFFFFFFFF é o maior valor possível e é muito maior que 1.

Observações

  • Realiza uma comparação sem sinal. Para comparação com sinal, use slt.
  • O resultado em `rd` é sempre 0 ou 1.
  • Essencial para aritmética de ponteiros e comparações de tamanhos ou contagens.
