SH - Armazenar Meia-Palavra (Store Halfword)

Sintaxe

sh rs2, imm(rs1)

Descrição

Armazena a meia-palavra menos significativa (16 bits) do registrador de origem rs2 na memória. O endereço de memória é calculado somando o deslocamento imediato imm ao valor no registrador base rs1.

Formato e Codificação

Formato: S-type
Opcode: 0100011
funct3: 0x1

Operação

mem[rs1 + imm] = rs2[15:0]

Exemplos de Uso

Exemplo 1: Escrever em um array de `short`

.data array_shorts: .space 8 # Aloca 8 bytes para 4 shorts .text la t0, array_shorts li t1, -10 # Valor a ser armazenado sh t1, 2(t0) # Escreve -10 no segundo elemento do array # (offset de 2 bytes)
`sh` é a instrução correta para armazenar valores de 16 bits, como um tipo `short` em C, em um array.

Exemplo 2: Modificar campo de 16 bits em uma struct

# Suponha que t0 aponta para o início de uma struct # e o campo que queremos modificar está no offset +4 li t1, 2024 # Novo valor para o campo (e.g., ano) sh t1, 4(t0) # Armazena o valor 2024 no campo da struct
Útil para manipulação de estruturas de dados onde campos individuais têm 16 bits de tamanho.

Observações

  • O endereço de memória (rs1 + imm) deve ser alinhado a 2 bytes. Endereços não alinhados podem causar uma exceção.
  • O deslocamento imm é um valor com sinal de 12 bits.
  • Apenas os 16 bits menos significativos de rs2 são escritos na memória.
  • É a instrução de armazenamento padrão para dados de 16 bits.
