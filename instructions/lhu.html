LHU - Carregar Meia-Palavra sem Sinal (Load Halfword Unsigned)

Sintaxe

lhu rd, imm(rs1)

Descrição

Carrega uma meia-palavra (16 bits) da memória para o registrador de destino rd. O endereço de memória é calculado somando o deslocamento imediato imm ao valor no registrador base rs1. A meia-palavra carregada é então estendida com zeros para 32 bits antes de ser armazenada em rd.

Formato e Codificação

Formato: I-type
Opcode: 0000011
funct3: 0x5

Operação

rd = zero_extend(mem[rs1 + imm])

Exemplos de Uso

Exemplo 1: Carregar de um array de `unsigned short`

.data array_ushorts: .half 0x8000, 0xFFFF .text la t0, array_ushorts lhu t1, 0(t0) # Carrega o primeiro elemento (0x8000) # t1 conterá 0x00008000 (32768) lhu t2, 2(t0) # Carrega o segundo elemento (0xFFFF) # t2 conterá 0x0000FFFF (65535)
`lhu` é usado para acessar dados de 16 bits que devem ser tratados como valores positivos (0-65535).

Exemplo 2: Diferença para LH

# Suponha que o endereço 0x1000 contém a meia-palavra 0x8000 li t0, 0x1000 lhu t1, 0(t0) # t1 = 0x00008000 (32768) - Extensão com zero lh t2, 0(t0) # Para comparação, lh resultaria em 0xFFFF8000 (-32768)
`lhu` preenche os 16 bits superiores com 0 (extensão de zero), tratando a meia-palavra como um número sem sinal. `lh` preencheria com 1, pois o bit 15 da meia-palavra é 1.

Observações

  • O endereço de memória (rs1 + imm) deve ser alinhado a 2 bytes. Endereços não alinhados podem causar uma exceção.
  • O deslocamento imm é um valor com sinal de 12 bits (-2048 a +2047).
  • Use `lhu` para dados de 16 bits que representam valores sem sinal (0 a 65535).
  • Para carregar dados de 16 bits com sinal, use a instrução lh.
