JAL - Salto e Liga (Jump and Link)

Sintaxe

jal rd, lab

Descrição

Armazena o endereço de retorno (PC+4) no registrador rd, então salta para o rótulo lab. Permite especificar qualquer registrador como destino do endereço de retorno.

Formato e Codificação

Formato: J-type
Opcode: 1101111
Alcance: ±1 MB (±2^20 bytes)
Offset: 20 bits, múltiplo de 2

Operação

rd = PC + 4 PC = PC + offset

Exemplos de Uso

Exemplo 1: Chamada com registrador customizado

main: li x10, 100 jal x5, processa # salva retorno em x5 ao invés de ra # ra permanece intacto para retorno de main add x11, x10, x5 processa: addi x10, x10, 50 jalr x0, x5, 0 # retorna usando x5
Útil quando ra já contém um endereço de retorno que precisa ser preservado.

Exemplo 2: Salvar múltiplos endereços de retorno

funcao_externa: addi sp, sp, -4 sw ra, 0(sp) # salva ra na pilha jal x6, funcao_A # usa x6 para retorno jal x7, funcao_B # usa x7 para retorno lw ra, 0(sp) # restaura ra addi sp, sp, 4 ret
Permite gerenciar múltiplos pontos de retorno sem usar a pilha para todos.

Exemplo 3: Salto sem salvar retorno

jal x0, destino # equivalente a "j destino" # não salva endereço de retorno (x0 sempre = 0) destino: # código continua aqui sem possibilidade de retornar
Usar x0 como rd descarta o endereço de retorno, transformando em salto incondicional.

Observações

  • Esta é a instrução real de máquina, não uma pseudo-instrução
  • Permite escolher qualquer registrador como destino (rd)
  • Se rd = x0, funciona como salto incondicional (equivalente a j)
  • Se rd = x1 (ra), é a forma padrão para chamadas de função
  • O offset é sempre múltiplo de 2 (alinhamento de instruções)
  • Alcance de ±1 MB permite saltos para a maioria das funções
  • Para distâncias maiores, use sequência auipc + jalr
  • Offset é calculado automaticamente pelo assembler
